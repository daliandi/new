## 功能背景

本项目构建一套基于 Spring Boot 2.2.x 的企业级 Web 管理平台，参考 RuoYi 系统设计，提供权限管理、组织架构、系统监控等完整基础能力，并承载无人机信息管理业务功能。

**当前痛点**：
- 缺乏统一的权限管控体系，用户、角色、菜单权限分散难以维护
- 无标准化的系统监控手段，CPU/内存/数据库连接池状态不可见
- 无人机资产信息依赖人工录入，属性填写繁琐且易出错

**预期收益**：
- 建立完整的 RBAC 权限体系，支持菜单权限和数据权限双维度管控
- 提供服务监控、缓存监控、连接池监视一体化运维视图
- 通过 AI 自动生成无人机属性，减少人工录入工作量

---

## 目标

**主要目标**：
1. 实现完整的用户/角色/菜单/部门/岗位权限管理体系
2. 实现系统运维监控（服务、缓存、连接池、在线用户、定时任务）
3. 实现无人机信息管理，支持 AI 自动属性生成
4. 支持 SQLite 与 MySQL 双数据库 Profile 切换
5. 遵循四层架构约束，service/repository 接口与实现分包

**成功指标**：
- 所有基础功能模块可正常使用：用户、部门、岗位、菜单、角色、字典、参数、公告、日志
- 系统监控页面可实时展示 CPU/内存/磁盘/JVM 数据
- 无人机 AI 属性生成成功率 ≥ 90%（网络正常情况下）
- 单元测试覆盖率 ≥ 80%

## 用户故事

### 故事 1：用户登录与认证

**As a** 系统用户  
**I want** 通过用户名和密码安全登录系统  
**So that** 我可以访问受权限保护的功能模块

#### 验收标准

- **Given** 用户在登录页输入正确的用户名和密码  
  **When** 点击登录按钮  
  **Then** 系统验证凭据，创建 Shiro Session，跳转至系统首页，并记录登录日志

- **Given** 用户输入错误的用户名或密码  
  **When** 点击登录按钮  
  **Then** 系统返回"用户名或密码错误"提示，记录失败次数

- **Given** 同一账号连续登录失败 5 次  
  **When** 再次尝试登录  
  **Then** 系统锁定账号 30 分钟，返回锁定提示

- **Given** 用户未登录  
  **When** 访问受保护页面  
  **Then** 系统重定向至登录页面

#### 异常场景

- **Given** 用户账号已被管理员禁用  
  **When** 尝试登录  
  **Then** 系统返回"账号已停用"提示，拒绝登录

---

### 故事 2：用户管理

**As a** 系统管理员  
**I want** 管理系统用户账号  
**So that** 我可以控制哪些人员可以访问系统及其权限范围

#### 验收标准

- **Given** 管理员在用户管理页面填写完整的用户信息  
  **When** 点击新增保存  
  **Then** 系统创建用户记录，密码以 MD5 加盐哈希存储

- **Given** 管理员输入已存在的用户名  
  **When** 点击新增保存  
  **Then** 系统返回"用户名已存在"错误提示

- **Given** 管理员在用户列表页设置筛选条件  
  **When** 点击查询  
  **Then** 系统返回符合条件的分页用户列表，每页默认 10 条

- **Given** 管理员选择一个用户并点击重置密码  
  **When** 确认操作  
  **Then** 系统将密码重置为系统默认密码（加盐哈希存储）

#### 异常场景

- **Given** 管理员尝试删除当前登录的自身账号  
  **When** 点击删除  
  **Then** 系统拒绝操作并返回"不能删除当前登录用户"提示

---

### 故事 3：部门管理

**As a** 系统管理员  
**I want** 维护公司/部门/小组的树形组织架构  
**So that** 我可以按组织层级管理用户和数据权限

#### 验收标准

- **Given** 管理员进入部门管理页面  
  **When** 页面加载  
  **Then** 系统以树形结构展示所有部门，支持展开和折叠

- **Given** 管理员填写部门信息并选择上级部门  
  **When** 点击新增保存  
  **Then** 系统创建部门节点并更新树形结构

#### 异常场景

- **Given** 管理员尝试删除存在子部门或关联用户的部门  
  **When** 点击删除  
  **Then** 系统拒绝删除并返回具体原因提示

---

### 故事 4：角色与权限管理

**As a** 系统管理员  
**I want** 配置角色的菜单权限和数据权限范围  
**So that** 我可以精细控制不同角色用户的功能访问和数据可见范围

#### 验收标准

- **Given** 管理员创建角色并勾选菜单权限树  
  **When** 点击保存  
  **Then** 系统保存角色与菜单的关联关系，该角色用户登录后仅显示已授权菜单

- **Given** 管理员为角色设置数据权限范围为"本部门及子部门"  
  **When** 该角色用户查询用户列表  
  **Then** 系统仅返回该用户所在部门及子部门的用户数据

---

### 故事 5：定时任务管理

**As a** 系统管理员  
**I want** 在线添加、修改和监控定时任务  
**So that** 我可以调度后台自动化作业并查看执行结果

#### 验收标准

- **Given** 管理员填写任务名称、调用目标和 Cron 表达式  
  **When** 点击新增保存  
  **Then** 系统注册定时任务，按 Cron 表达式调度执行

- **Given** 定时任务执行完成  
  **When** 查看执行日志  
  **Then** 系统展示执行时间、耗时、执行状态和异常信息（如有）

#### 异常场景

- **Given** 定时任务执行时抛出异常  
  **When** 任务执行完成  
  **Then** 系统将执行状态标记为失败，记录异常信息，不影响其他任务继续执行

---

### 故事 6：无人机信息管理

**As a** 业务操作员  
**I want** 录入无人机信息并由 AI 自动生成属性  
**So that** 我可以快速建立无人机资产台账，减少手动填写工作量

#### 验收标准

- **Given** 操作员输入无人机编号  
  **When** 触发 AI 属性生成  
  **Then** 系统调用 AIService 自动填充型号、制造商、最大载重、最大飞行时间、最大飞行速度、最大飞行高度、用途分类等属性

- **Given** 操作员在无人机列表页设置筛选条件  
  **When** 点击查询  
  **Then** 系统返回符合条件的分页无人机列表，每页默认 10 条

- **Given** 操作员选择多条无人机记录  
  **When** 点击批量删除  
  **Then** 系统删除所选记录并刷新列表

#### 异常场景

- **Given** AIService 调用超时或返回错误  
  **When** 触发 AI 属性生成  
  **Then** 系统返回"AI 生成失败，请手动填写"提示，允许操作员手动录入属性

---

### 故事 7：服务监控

**As a** 系统管理员  
**I want** 查看服务器和 JVM 运行状态  
**So that** 我可以及时发现资源瓶颈和异常

#### 验收标准

- **Given** 管理员进入服务监控页面  
  **When** 页面加载  
  **Then** 系统展示 CPU 使用率、内存使用情况、JVM 堆内存、磁盘分区信息和操作系统信息

---

### 故事 8：代码生成器

**As a** 开发者  
**I want** 根据数据库表结构自动生成 CRUD 代码骨架  
**So that** 我可以快速搭建新业务模块，减少重复编码

#### 验收标准

- **Given** 开发者选择数据库表并配置包路径和模块名  
  **When** 点击生成  
  **Then** 系统生成 domain 实体、Mapper 接口、Mapper XML、Service 接口、Service 实现、Controller 和 Thymeleaf 页面，打包为 ZIP 下载

- **Given** 开发者点击预览  
  **When** 预览页面加载  
  **Then** 系统展示各文件的生成内容，供确认后再下载

## 非功能性需求

### 性能要求

- **响应时间**：普通列表查询接口响应时间 ≤ 500ms（正常负载，数据量 ≤ 10 万条）
- **并发量**：支持 50 并发用户同时操作
- **数据量**：操作日志表支持 100 万条记录的分页查询

### 安全要求

- **认证**：所有业务接口需通过 Apache Shiro 认证，未登录请求重定向至登录页
- **授权**：接口级权限通过 Shiro 注解（`@RequiresPermissions`）控制，按钮权限通过权限标识控制
- **数据保护**：用户密码使用 MD5 加盐哈希存储，禁止明文传输和存储
- **输入验证**：所有用户输入通过 Hibernate Validation 进行参数校验，防止 SQL 注入（MyBatis 参数化查询）
- **XSS 防护**：系统对富文本输入进行 XSS 过滤

### 兼容性要求

- **浏览器兼容性**：支持 Chrome 80+、Firefox 75+、Edge 80+
- **数据库兼容**：支持 SQLite 3.x 和 MySQL 5.7+，通过 Spring Profile 切换
- **Java 版本**：Java EE 8，运行时 JDK 8+

### 可用性要求

- **故障恢复**：定时任务执行失败不影响系统其他功能正常运行
- **日志记录**：所有异常均记录至日志文件，便于问题排查

## 架构约束

- 遵循四层架构：`controller → service → domain ← repository`
- service 接口放在 `service` 包，实现放在 `service.impl` 包
- repository（Mapper）接口放在 `repository` 包
- 所有拦截器放在独立的 `interceptor` 包
- 尽量使用注解，减少 XML 配置文件依赖
- 全局异常处理器统一返回 `{"code": int, "msg": string, "data": object}` 格式

- 移动端（App/小程序）适配

- 消息队列集成（MQ）

- 视图层迁移至 Vue + Element UI
- 引入 Redis 替代本地缓存
- 支持 OAuth2 第三方登录
