## 功能背景与目标

本系统是一个基于 Spring Boot 2.2.x + Apache Shiro + MyBatis 技术栈构建的无人机信息管理平台，
参考 RuoYi 系统的基础功能设计风格，提供无人机信息的录入、查询、删除和修改能力。
无人机属性支持由 AI 自动生成，支持 SQLite 和 MySQL 双数据库无缝切换。

**核心目标**：
- 提供完整的无人机 CRUD 管理能力
- 支持 SQLite（开发/轻量级场景）与 MySQL（生产场景）自由切换
- 以 Thymeleaf 模板引擎渲染 Web 页面，采用 Bootstrap 3.3.7 风格
- AI 自动生成无人机属性，减少手工录入负担

## 用户故事

### US-01：录入无人机信息

**As a** 管理员
**I want to** 录入新无人机的基本信息（支持 AI 自动生成属性）
**So that** 系统中存在可追踪、可管理的无人机档案

**验收标准（Given/When/Then）**：
- Given 管理员已登录系统
- When 管理员点击"新增"并填写无人机信息（或点击"AI 生成"）
- Then 系统保存无人机信息，列表页新增一条记录，并提示"新增成功"

### US-02：查询无人机信息

**As a** 管理员
**I want to** 按条件（型号、状态、注册号等）搜索无人机列表
**So that** 快速定位目标无人机档案

**验收标准**：
- Given 系统中存在若干无人机记录
- When 管理员输入搜索条件并提交
- Then 列表显示符合条件的无人机，支持分页，关键字高亮

### US-03：修改无人机信息

**As a** 管理员
**I want to** 修改已存在的无人机属性
**So that** 保持档案数据准确

**验收标准**：
- Given 目标无人机记录已存在
- When 管理员点击"编辑"、修改字段后提交
- Then 数据库更新对应字段，列表显示最新信息，提示"修改成功"

### US-04：删除无人机信息

**As a** 管理员
**I want to** 删除无效的无人机记录（逻辑删除）
**So that** 列表中只显示有效档案

**验收标准**：
- Given 目标无人机记录已存在
- When 管理员点击"删除"并二次确认
- Then 记录被逻辑删除（`deleted=1`），列表不再显示该记录

### US-05：AI 自动生成无人机属性

**As a** 管理员
**I want to** 通过 AI 自动生成无人机的规格属性（最大载重、飞行高度、续航时间等）
**So that** 减少手动录入负担，提升录入效率

**验收标准**：
- Given 管理员打开新增页面
- When 点击"AI 生成属性"按钮
- Then 系统自动填充无人机属性字段（可供管理员二次修改）

## 非功能性需求

| 类别 | 要求 |
|------|------|
| 数据库 | 支持 MySQL 5.7+ 和 SQLite 3.x，通过配置文件切换，无需修改代码 |
| 安全 | 基于 Apache Shiro 1.7，支持登录认证、权限拦截 |
| 性能 | 列表查询响应时间 < 500ms（10万条数据量） |
| 可维护性 | 遵循四层架构约束，Service/Mapper 接口与实现分包 |
| 编码规范 | 优先注解配置，最小化 XML 依赖；拦截器独立包管理 |
| 页面风格 | 采用 Bootstrap 3.3.7，参考 RuoYi 页面布局 |

---

## 排除范围

- 不包含用户注册功能（管理员账号由初始化脚本创建）
- 不包含无人机实时飞行轨迹/遥测数据管理
- 不包含多租户隔离
